CloudLens Studio is a full-stack, serverless image processing application. It allows users to securely upload, process (resize, filter, rotate, watermark), and store images in the cloud using an event-driven AWS architecture.

## ✨ Key Features
* **Secure Authentication:** User login and session management powered by Clerk.
* **Direct-to-Cloud Uploads:** Secure, pre-signed URLs generated by the Node.js backend to upload files directly to an AWS S3 Raw Bucket.
* **Event-Driven Processing:** An AWS Lambda worker automatically triggers upon upload, processes the image using Sharp, and saves it to a Processed Bucket.
* **Persistent Gallery:** Processed image URLs and metadata are stored in MongoDB and served via a paginated gallery interface.
* **Raw Database Management:** Integrated functionality to clean up the raw S3 bucket to manage cloud storage costs.

## 🚀 Local Setup & Installation

If you want to run this project locally, follow these steps:

### 1. Clone the repository
```bash
git clone [https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git)
cd YOUR_REPO_NAME
```

### 2. Set up Environment Variables
Because this app relies heavily on cloud services, you will need to create a `.env` file in the `backend` directory. **Do not commit this file to version control.**

```env
# backend/.env
PORT=5000
MONGO_URI=your_mongodb_connection_string
AWS_REGION=your_aws_region
AWS_ACCESS_KEY_ID=your_aws_access_key
AWS_SECRET_ACCESS_KEY=your_aws_secret_key
AWS_BUCKET_NAME=your_raw_s3_bucket_name
AWS_PROCESSED_BUCKET_NAME=your_processed_s3_bucket_name
```

*(Note: You will also need to configure Clerk environment variables in your frontend directory according to Clerk's documentation).*

### 3. Install Dependencies
You will need to install dependencies for both the frontend and backend.

```bash
# Install backend dependencies
cd backend
npm install

# Install frontend dependencies
cd ../frontend
npm install
```

### 4. Run the Application
Open two terminal windows:

**Terminal 1 (Backend):**
```bash
cd backend
npm run dev
```

**Terminal 2 (Frontend):**
```bash
cd frontend
npm run dev
```

The app will now be running on `http://localhost:5173`.
